Real-Time Weapon
Detection System

"Can a multi-model deep learning ensemble ‚Äî combining YOLO variants (v5, v8, v11) with custom PyTorch and Keras CNNs ‚Äî outperform single-model approaches in real-time weapon detection accuracy, recall, and confidence for public surveillance applications?"

Problem Statement

Conventional surveillance systems rely heavily on human monitoring ‚Äî a process that is error-prone, non-scalable, and slow to respond. With rising incidents of public violence, there is a critical need for an automated, real-time system capable of detecting weapons such as guns and knives from live camera feeds and static images. This project addresses that gap: reducing response time, assisting law enforcement, and promoting safer public spaces through early, automated threat detection.

Literature Review / Market Research

Object detection has evolved from Faster R-CNN (2015) through YOLO v1‚Äìv12, with each generation improving the speed-accuracy tradeoff. Redmon et al. (CVPR 2016) introduced single-pass detection; Wang et al. (IEEE TPAMI 2020) provided comprehensive deep learning survey benchmarks. Existing deployed weapon detection systems typically rely on a single YOLO variant with no cross-model consensus or agreement logic.

Research Gap / Innovation

Existing literature lacks comparative analyses of lightweight vs. heavyweight models in real-time weapon classification. This project innovates by deploying a 6-model ensemble ‚Äî YOLOv5, YOLOv8, YOLOv11, PyTorch CNN, Kaggle CNN, and Keras CNN ‚Äî with inter-model agreement logic (majority vote) that reduces false positives. No current state-of-the-art solution combines all six architectures in a unified FastAPI + WebSocket pipeline.

System Methodology
Dataset / Input

Custom labeled weapon image dataset (guns & knives) supplemented with the Kaggle weapon detection dataset. Preprocessing: resize to 640√ó640, normalization, augmentation (flip, rotate, brightness). Supports .jpg/.png static uploads and live webcam WebSocket streams via FastAPI.

Model / Architecture

6-model ensemble: YOLOv5s, YOLOv8s, YOLOv11s via Ultralytics for bounding box detection; PyTorch CNN binary classifier; Kaggle multi-class CNN; Keras CNN. FastAPI backend (REST + WebSocket). Inter-model majority vote reduces false positives. Inference target: ‚â§1s per image.
